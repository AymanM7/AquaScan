# PHASE 08 — Reports, Inbox & Dealroom
## Pages: `/report/[id]`, `/inbox`, `/dealroom/[id]`

**Prerequisite:** Phases 01–07 complete. Automation engine generating reports. Auth0 roles configured.

---

## 1. Objective

Build three auth-gated, data-dense pages that complete the "last mile" of the autonomous prospecting workflow:

- **`/report/[id]`** — Full classified-style intelligence dossier from one automation run
- **`/inbox`** — Rep notification inbox showing all routed reports
- **`/dealroom/[id]`** — Shareable, printable evidence package with Auth0 human-in-the-loop approval gate

These pages are the "trust layer" — they must look professional enough that a sales rep would forward them to their manager without modification.

---

### Human-in-the-Loop Approval Gate — `components/shared/ApprovalGate.tsx`

This is the key Auth0 AI Agents demonstration:

```tsx
// "Send This Dossier to Prospect" button at bottom of dealroom
// On click: opens ApprovalGate modal

interface ApprovalGateProps {
  buildingName: string
  recipientEmail: string
  recipientName: string
  emailSubject: string
  emailPreview: string  // first 100 chars of generated email
  onApprove: () => void
  onReject: () => void
  onEdit: () => void
}

// Modal:
<ApprovalGate.Root open={gateOpen}>
  <ApprovalGate.Overlay />
  <ApprovalGate.Content>
    
    {/* Header */}
    <div className="gate-header">
      <span className="gate-ai-badge">🤖 AI Agent Action Pending</span>
      <h3>Review Before Sending</h3>
    </div>
    
    {/* Action description */}
    <div className="gate-action">
      <p>Your AI agent is ready to send this dossier to:</p>
      <div className="gate-recipient">
        <strong>{recipientName}</strong>
        <span>{recipientEmail}</span>
      </div>
      <div className="gate-email-preview">
        <p className="gate-email-subject">Subject: {emailSubject}</p>
        <p className="gate-email-body">{emailPreview}...</p>
      </div>
    </div>
    
    {/* Action log */}
    <div className="gate-action-log">
      <p className="gate-log-label">Action log:</p>
      <p>• AI drafted personalized dossier email using building data and Perplexity research</p>
      <p>• Recipient verified: {recipientName} ({recipientEmail})</p>
      <p>• Action: Send email from grundfos-rep@grundfos.com</p>
      <p className="gate-waiting">⏸ Waiting for human authorization...</p>
    </div>
    
    {/* Action buttons */}
    <div className="gate-actions">
      <button className="gate-approve" onClick={handleApprove}>✓ Approve & Send</button>
      <button className="gate-edit" onClick={handleEdit}>✎ Edit First</button>
      <button className="gate-reject" onClick={handleReject}>✕ Reject</button>
    </div>
    
  </ApprovalGate.Content>
</ApprovalGate.Root>
```

**On Approve:**
1. Log the action: `POST /api/inbox/{notification_id}/action` with `{ action_type: 'approved' }`
2. For demo: show success toast "Dossier sent to [Company Name]"
3. In production: trigger an email send via SendGrid or similar
4. Update notification `actioned_at` and `action_type` in DB

**On Reject:**
1. Log the action with `action_type: 'rejected'`
2. Modal closes
3. Show toast "Action declined. Report saved for future reference."

The visual effect of this modal on demo judges is powerful: they see an AI system that assembled an email action, then visibly paused and waited for a human to authorize it. This is enterprise-grade AI governance demonstrated in 10 seconds.
